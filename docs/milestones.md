# AIdea · 里程碑与任务拆分

## 1. 文档目标

本文档将 `AIdea` 的实施工作拆分为阶段性里程碑、验收标准与开发任务，便于按顺序推进落地。

## 2. 交付策略

首版采用“`先搭骨架，再完成学习系统主链路，最后补齐质量与部署`”的实施方式：

1. 先完成工程初始化与基础架构
2. 再完成用户、学习路径、知识内容、成长体系的主业务闭环
3. 最后补充 Dashboard、质量保障与部署文档

## 3. 总体里程碑

- `M1`：工程骨架完成
- `M2`：学习系统基础可用
- `M3`：AI Agent 学习主链路可用
- `M4`：可部署版本完成

## 4. M1 工程骨架完成

### 目标

建立可运行的 monorepo、前后端基础工程和配置驱动基础设施。

### 任务清单

- 初始化本地 Git 仓库
- 建立 `.gitignore`
- 约定主分支开发规范与分支命名规范
- 初始化根目录 `package.json`
- 建立 `npm workspaces`
- 初始化 `apps/client`
- 初始化 `apps/api`
- 初始化 `packages/shared`
- 初始化 `packages/config`
- 建立 `tsconfig.base.json`
- 建立 `eslint` 与 `prettier` 配置
- 建立环境变量约定
- 建立 `uni-app H5` 启动链路
- 建立 `NestJS` API 启动链路
- 建立配置 schema 与配置加载机制
- 建立测试目录约定与测试运行脚本
- 约定测试驱动开发流程：先写用例，再开发，再验证

### 验收标准

- `npm install` 可成功执行
- 本地 Git 仓库可正常工作
- 前端 H5 可本地启动
- 后端 API 可本地启动
- 至少一类页面模块可通过配置渲染
- 基础 README 和文档入口已建立

## 5. M2 学习系统基础可用

### 目标

实现学习资料组织、个性化学习偏好与知识管理的最小业务闭环（**不强制登录**；默认用户 admin，见 `docs/m2/design-auth-reserved.md`）。

### 任务清单

- ~~完成用户注册与登录~~（本阶段不开放注册；**不做 JWT 业务校验**，保留 `POST /auth/login` 供扩展）
- 建立角色模型
- 建立用户学习偏好配置
- 支持用户选择学习方向偏好，首期默认突出 `AI Agent`
- 完成知识条目 CRUD
- 完成分类与标签管理
- 完成笔记 CRUD
- 实现笔记关联知识条目
- 实现知识列表筛选与搜索
- 实现知识详情页与笔记详情页
- 将知识表单和列表的一部分能力配置化
- 建立学习内容类型与难度等级配置
- 建立学习路径配置骨架

### 实施说明（以设计为准）

迭代范围与鉴权策略可能随 `docs/m2/README.md` 调整（例如：**默认 admin、无 JWT 校验**、笔记延后）；**交付与验收以该目录下最新设计文档为准**，本节条目标题为原始规划。

### 验收标准

> 实际验收以 `docs/m2/` 目录下设计文档为准（见「实施说明」）。

- 无需登录流程即可使用业务 API（后端绑定 **admin**）；**`POST /auth/login` 仍可调用**（预留）
- 用户可完成学习偏好设置，偏好持久化
- 用户可新建、编辑、删除知识条目
- 知识条目可绑定标签和分类
- 知识列表支持按分类/标签筛选与搜索
- 系统可依据用户偏好展示差异化学习入口（`/learning/entry`）
- ~~用户可完成注册~~（本阶段不开放注册，见实施说明）
- ~~笔记 CRUD~~（整体移到下一阶段，见 `design-notes-deferred.md`）

## 6. M3 AI Agent 学习主链路可用

### 目标

完成以 `AI 基础知识学习 -> AI Agent 定向路径 -> 成长升级反馈` 为主的学习闭环。

### 任务清单

- 设计学习路径实体与配置结构
- 建立学习阶段、模块、课程单元的数据结构
- 首期提供 `AI 基础认知`、`Prompt 基础`、`LLM 应用认知`、`AI Agent 入门` 四类学习模块
- 重点实现 `AI Agent` 方向学习路径
- 支持用户在设置中切换学习偏好方向
- 依据学习偏好生成首页推荐学习路径与模块排序
- 设计并实现成长等级体系
- 支持通过学习行为累计经验值与等级提升
- 完成等级文案配置，首期采用修仙风格等级，例如 `炼气`、`筑基`、`金丹`
- Dashboard 展示当前等级、经验进度、已完成学习模块与推荐下一步
- 记录学习行为日志，用于等级计算和进度展示

### 验收标准

- 用户可选择学习偏好，并看到偏向 `AI Agent` 的学习路径
- 用户可按路径完成学习模块并标记进度
- 用户完成学习后经验值会增长并触发等级变化
- Dashboard 可展示等级、进度和推荐学习内容

## 7. M4 可部署版本完成

### 目标

补齐测试、部署说明和上线前基础保障，并为后续扩展更多学习方向留出空间。

### 任务清单

- 编写关键单元测试
- 编写关键集成测试
- 编写核心 E2E 用例
- 完成 H5 构建文档
- 完成后端部署文档
- 增加 Docker 支持
- 梳理环境变量说明
- 输出移动端发布准备说明
- 输出新增学习方向的配置扩展示例
- 输出成长体系运营配置说明
- 建立测试基线与最小回归用例集
- 补充 GitHub 协作与提交流程说明

### 验收标准

- 核心学习流程具备自动化测试
- H5 版本可构建部署
- 后端可在标准 Node.js 或 Docker 环境部署
- 文档可支持后续持续开发
- 开发流程可按 `测试先行 -> 开发实现 -> 用例验证` 执行

## 8. 首批任务优先级

- 根目录工程初始化
- Git 仓库与基础版本管理
- `client` 与 `api` 启动
- 用户注册登录
- 用户学习偏好设置
- 学习路径配置系统
- AI 基础知识内容管理
- AI Agent 方向学习模块
- 等级成长体系
- 知识条目 CRUD
- 笔记 CRUD

## P1

- Dashboard
- 搜索筛选增强
- 学习推荐排序
- 学习行为日志展示
- 等级勋章与成就展示

## P2

- 分享页
- 多人协作增强
- AI 对话
- 实验记录
- RAG
- 原生端专项能力

## 9. 推荐开发顺序

1. 工程初始化
2. 鉴权
3. 学习偏好与学习路径配置
4. 知识与笔记
5. AI Agent 学习模块
6. Dashboard
7. 测试与部署

## 10. 人天预估

以下预估按单人开发、兼顾学习与实现节奏进行保守估计：

- `M1`：3 到 5 天
- `M2`：5 到 7 天
- `M3`：6 到 9 天
- `M4`：3 到 5 天

总计建议：`17 到 26 天`

## 11. 风险控制点

- 不在 M1 阶段追求过度完善的配置 DSL
- 不在 M2 阶段提前引入复杂多人协作
- 不在 M3 阶段同时实现 AI 对话和学习体系，先把学习主链路做完整
- 学习等级体系要可配置，避免把数值规则写死在页面里
- AI Agent 路径首期聚焦入门和岗位导向，不延伸到过深的底层模型训练知识
- 测试驱动开发要控制颗粒度，优先补高价值用例，避免大量低价值样板测试
- 不在 M4 阶段同时追求 H5 和移动端全量发布，先确保 H5 稳定

## 12. 开发完成定义

某个里程碑被视为完成，需要同时满足：

- 功能达到对应验收标准
- 代码可运行
- 关键文档已更新
- 无阻塞性已知问题
