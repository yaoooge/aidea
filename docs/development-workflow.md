# AIdea 开发流程

## 1. 版本管理

- 使用 `Git` 进行本地版本管理
- 远程仓库托管在 `GitHub`
- 优先小步提交，保持提交粒度清晰
- 新功能优先使用功能分支开发

## 2. 测试驱动开发

每个功能都按下面顺序推进：

1. 先写测试用例
2. 先运行测试，确认用例失败
3. 再实现功能
4. 再运行测试，确认通过

## 3. 测试范围

- 单元测试：配置解析、等级计算、学习路径推荐
- 集成测试：登录、偏好设置、学习模块完成
- E2E：核心学习主链路

## 4. 提交前检查

- 测试通过
- 关键文档已更新
- 无明显 lint 或类型错误

## 5. 当前一期重点

- 学习路径
- 知识条目
- 学习笔记
- 成长等级
- Dashboard
