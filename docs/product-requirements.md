# AIdea PRD

## 1. 文档目标

本文档用于细化 `AIdea` 的首版产品需求，明确 MVP 页面结构、核心用户流程、功能边界与配置化设计重点，作为前后端开发、数据建模和迭代规划的直接输入。

## 2. 产品定义

`AIdea` 是一个面向个人学习成长、同时预留多人扩展能力的跨端学习工具。首版以 `uni-app H5` 形态优先落地，聚焦以下两个核心方向：

- `AI 应用开发学习`：通过学习路径、知识模块、笔记和成长反馈帮助用户系统理解 AI，重点偏向 `AI Agent` 方向。
- `知识管理`：围绕学习资料、笔记、标签、知识条目建立结构化知识库。

## 3. 目标用户

### 3.1 当前主用户

- 个人学习者
- AI 技术实践者
- 需要长期沉淀学习资料与实验记录的开发者

### 3.2 后续扩展用户

- 小规模协作者
- 学习伙伴
- 内容分享对象

## 4. 产品目标

### 4.1 首版目标

- 建立可用的学习闭环：`学习偏好选择 -> 学习路径推荐 -> 知识学习 -> 笔记沉淀 -> 进度反馈 -> 等级提升`
- 提供统一的学习路径、知识条目、笔记、成长体系入口
- 具备基础用户体系，为多人扩展预留空间
- 通过配置驱动方式提高前端页面与功能扩展效率
- 支持用户按偏好选择学习方向，首期优先突出 `AI Agent`
- 建立可配置的等级成长体系，增强长期学习动力

### 4.2 非目标

- 首版不做复杂协作编辑
- 首版不做向量检索和深度 RAG
- 首版不做推荐系统
- 首版不接入原生端专项能力，如推送、离线同步、原生插件
- 首版不实现 AI 对话与实验记录主流程

## 5. 产品原则

- 围绕学习路径建设，不做泛聊天产品
- 优先实现高频核心路径，不追求一次性做全
- 配置优先，代码兜底
- H5 优先跑通，移动端后续渐进适配
- 多人架构预留，但体验优先服务个人学习场景
- 优先帮助用户建立 AI 求职导向的学习结构，而不是追求底层理论覆盖面

## 6. MVP 信息架构

首版信息架构如下：

- `Dashboard`
- `Learning Path`
- `Knowledge`
- `Notes`
- `Growth`
- `Auth`
- `Settings`

## 7. 页面需求

## 7.1 Dashboard

### 页面目标

让用户一进入系统就能看到当前学习状态、推荐路径、成长等级和待学习内容。

### 页面模块

- 学习概览卡片
- 当前等级与经验进度
- 推荐学习路径
- 最近知识条目
- 最近笔记
- 学习目标进度
- 下一步学习建议

### 核心功能

- 展示当前用户的总知识条目数、笔记数、完成模块数
- 展示最近 7 天活跃情况
- 快速跳转到“继续学习”“新建知识条目”“新建笔记”
- 支持通过配置决定卡片是否显示、显示顺序、标题文案

### 配置化重点

- Dashboard 卡片注册表
- 卡片排序配置
- 首页快捷入口配置

## 7.2 Knowledge

### 页面目标

构建结构化知识条目库，承载学习资料的整理与沉淀。

### 页面组成

- 知识条目列表页
- 知识条目详情页
- 知识条目编辑页

### 字段要求

- 标题
- 摘要
- 正文内容
- 来源链接
- 来源类型
- 分类
- 标签
- 关联笔记
- 可见范围

### 核心功能

- 新建、编辑、删除知识条目
- 列表筛选：分类、标签、来源类型、关键词
- 详情页查看关联笔记与所属学习模块

### 配置化重点

- 列表列定义配置
- 表单字段配置
- 来源类型配置
- 详情区块配置

## 7.3 Notes

### 页面目标

帮助用户将学习过程中的理解、拆解和总结沉淀为可检索的结构化笔记。

### 页面组成

- 笔记列表页
- 笔记详情页
- 笔记编辑页

### 核心功能

- 新建学习笔记
- 关联知识条目
- 使用模板快速创建笔记
- 支持使用笔记模板记录知识理解、面试要点和实践总结

### 配置化重点

- 笔记模板配置
- 编辑器工具栏配置
- 详情页工具区配置

## 7.4 Learning Path

### 页面目标

帮助用户基于偏好选择适合自己的学习方向，并围绕 `AI Agent` 优先构建结构化学习路径。

### 页面组成

- 学习方向选择页
- 学习路径总览页
- 学习模块详情页
- 学习进度页

### 核心功能

- 首次使用时选择学习偏好
- 支持选择 `AI Agent` 作为主方向
- 根据偏好展示模块顺序和推荐内容
- 展示每个模块的学习目标、前置条件和完成状态
- 支持模块完成打卡

### 配置化重点

- 学习方向配置
- 学习路径配置
- 学习模块配置
- 模块排序与解锁条件配置

## 7.5 Growth

通过等级、经验值、阶段称号和成就反馈，让用户在长期学习中获得持续正反馈。

### 配置化重点

- 等级阈值配置

- 成长文案配置
- 成就规则配置

## 8. 用户核心流程

## 8.1 学习闭环主流程

1. 用户首次进入系统并选择学习偏好
2. 系统推荐以 `AI Agent` 为主的学习路径
3. 用户学习某个模块并查看对应知识条目
4. 用户创建或补充学习笔记
5. 用户完成模块打卡，系统记录进度与经验值
6. Dashboard 自动展示当前等级、进度与推荐下一步

## 8.2 学习路径配置流程

1. 管理端定义学习方向配置
2. 为方向绑定模块、顺序和前置条件
3. 为模块绑定知识条目和笔记模板
4. 前端根据用户偏好和配置动态渲染路径

## 8.3 知识沉淀流程

1. 用户新建知识条目
2. 录入内容与来源
3. 打标签和分类
4. 关联所属学习模块和已有笔记或新建笔记
5. 在详情页查看知识条目全景信息

## 9. 配置驱动设计范围

以下内容优先采用配置文件声明：

- 页面导航
- Dashboard 卡片
- 学习方向
- 学习路径
- 学习模块
- 列表字段
- 表单 schema
- 知识来源类型
- 等级体系
- 设置项定义
- 页面空状态文案

以下内容不建议在首版完全配置化：

- 复杂业务流程编排
- 权限判断核心逻辑
- 大段富文本编辑器内部行为

## 10. 功能边界

### P0

- 用户注册/登录
- 学习偏好设置
- 学习路径配置与展示
- AI Agent 方向学习模块
- 等级成长体系
- 知识条目 CRUD
- 笔记 CRUD
- Dashboard 基础卡片

### P1

- 搜索筛选增强
- Dashboard 更多统计
- 学习推荐增强
- 成就体系

### P2

- 分享页
- 协作能力细化
- AI Chat
- 实验记录
- RAG 与高级检索
- 原生端专项能力

## 11. 验收标准

- 用户能够完成从“选择学习偏好”到“学习模块完成”再到“等级提升”的完整闭环
- 首版所有核心页面在 H5 中可正常访问
- 页面可通过配置控制至少一类模块渲染
- 系统具备基础登录能力与角色区分
- 关键数据可通过后台 API 正常读写
